### Slide 2 — A2A protocol architecture

**Discovery:**
- Client does `GET` agent card (typically `/.well-known/agent-card.json`)
- Agent Card JSON: name, capabilities, skills, security schemes, API URL

**Communication:**
- JSON-RPC messages with `Role` (user/agent) and `Parts` (text, file, json)
- **`POST /sendMessage`** — creates a **Task**, returns Task Response
- **`POST /sendMessageStream`** — SSE stream: `Task (Submitted)` → `TaskStatusUpdateEvent (Working)` → `TaskArtifactUpdateEvent` → `Completed`
- Client can poll task status for long-running work

**Orchestrator pattern:**
- Client Agent (orchestrator) selects the right Remote Agent
- Example: flight task → flight-agent, calendar task → calendar-agent

**Core entities:**
- **Task** — central object (status, message history, artifacts)
- **Message** — unit of communication
- **Part** — text / file / URL / JSON inside a message
- **Artifact** — result output from a task

**Auth (in slide, but you said skip it):**
- Full protocol supports OpenID Connect + JWT
- Your requirement: **без авторизации** — implement discovery + `sendMessage` without JWT/OAuth
